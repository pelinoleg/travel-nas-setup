# Dashboard V2 — Визуальный язык и layout

Документ-парковка визуальной концепции UI для web-kiosk на Waveshare 4.3″ DSI 800×480 IPS (после приезда). НЕ про технологии — только про то **как организовать, что куда положить, как это выглядит**. Чисто визуал.

См. также [V2-WEB-DASHBOARD.md](V2-WEB-DASHBOARD.md) — там архитектура и feature pool.

## 2. Рекомендация — гибрид C + A

**Status sidebar 220px слева** (всегда) + **content area** справа с **bottom tabs** внутри content.

```

│ throttle ok  │                                                     │
│              │                                                     │

│ Personal ok  │                                                     │
│ 2h 14m ago   │                                                     │
│              │                                                     │

## 4. Визуальный язык — design tokens

### Цвета (тёмная тема)
```
bg          #0a0a0c    почти-чёрный, всё что не карточка
card        #16181d    карточки и панели
border      #2a2d35    тонкие разделители 1px
text-1      #ECEDEE    основной текст
text-2      #B3B7BD    вторичный текст
text-3      #787C84    подписи / muted

accent      #4ade80    зелёный — ok / running
warn        #fbbf24    амбер — внимание
error       #f87171    красный — проблема
info        #60a5fa    синий — действия / ссылки
purple      #a78bfa    Tailscale / VPN брендирование
```

### Типографика
Один шрифт — **Inter** (или IBM Plex Sans). Тариф размеров:
```
11pt   подписи карточек, time-stamps                    weight 400
14pt   body, списки, кнопки                              weight 400/500
18pt   заголовки карточек, section labels                weight 600
28pt   главные числа (76% · 47°C)                        weight 700
40pt   только в progress mode (% на пол-экрана)          weight 800
caps   = uppercase + letter-spacing 0.1em — для labels
```

### Радиусы и тени
```
radius        12px на карточках, 8px на кнопках, 999px на pill-tabs
shadow-card   0 1px 3px rgba(0,0,0,0.4)
shadow-modal  0 8px 32px rgba(0,0,0,0.6)
```

### Spacing
```
xs 4px   sm 8px   md 12px   lg 16px   xl 24px   2xl 32px
```
Между карточками всегда **md (12px)**. Внутри карточки padding `lg (16px)`. Между tabs **xl (24px)**.

## 5. Микро-паттерны взаимодействий

| Жест | Действие |
|---|---|
| **Tap card** | Открыть детальную страницу |
| **Long-press card** | Quick-action menu (Backup → Run/Stop/Diff) |
| **Tap status-точку** | Показать reasoning («ok потому что rsync exit=0 для 5/5») |
| **Pull-to-refresh** | Forced refresh + анимация cosmos |
| **Swipe left/right в content** | Между вкладками (если sidebar модерации не использует свайпы) |
| **Tap прогресс-плашке в sidebar** | Переход на progress детальную |
| **Toast снизу справа** | После действия — «Backup started», 2 сек, исчезает |
| **Modal slide-up снизу** | Confirm dialogues (reboot/shutdown/flip) |
| **Empty-state иллюстрация** | Если нет данных — не пустота, а понятная подсказка с CTA |

### Статус-точки (анимация)
```
● ok      без анимации, ровный цвет
● warn    без анимации
● error   мягкий pulse: opacity 1.0 → 0.6 → 1.0, цикл 2s
● running синий + быстрый pulse 1s
○ idle    серый, без анимации
```

### Progress-индикаторы
- **Бар горизонтальный** — стандарт для процентов
- **Круговой** только когда нет % — спиннер на «calculating…»
- **Marquee бар** — когда прогресс есть но процент неизвестен (например, scanning)
- **Sparkline в карточке** — мини-история последних 24ч маленькой линией под основным числом

│                  T7 76% · all ok · last backup 2h ago             │
│                                                                    │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```
- Часы 96pt, дата 18pt
- Сводка одной строкой 14pt снизу
- Brightness тускнеет до 30% (если DSI PWM поддерживает)
- Тап в любое место → пробуждение в Home

Альтернативные ambient-режимы (выбирать в Settings):
- **Clock** (выше) — часы + сводка
- **Photo slideshow** — последние 50 фото из `usb-imports/` каждые 8 сек
- **Status mural** — карточки расплываются крупнее, без активных кнопок
- **Off** — просто чёрный

## 7. Empty states (когда нет данных)

Вместо «(no data)» — рисованная подсказка с CTA-кнопкой:

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│                          ╭───────╮                                 │
│                          │  📦   │   нет бэкапов                   │
│                          │       │   ещё не запускались            │
│                          ╰───────╯                                 │
│                                                                    │
│                ┌──────────────────────────────────┐                │
│                │       🚀  ЗАПУСТИТЬ ПЕРВЫЙ        │                │
│                └──────────────────────────────────┘                │
│                                                                    │
│                  или настрой через `/etc/travel-nas/`              │
└────────────────────────────────────────────────────────────────────┘
```

Похожие empty-states для:
- Photos (нет фото) → «Подключи SD-карту чтобы начать»
- Tailscale (down) → «Авторизуйся» + кнопка
- Verify (никогда не запускался) → «Запустить первый scrub» + кнопка

## 8. Что НЕ делать (anti-patterns)

- **Не делать вертикальные подразделы внутри карточек** — карточка одна цельная single-purpose. Иначе scrap на 3×2 grid.
- **Не использовать «гамбургер» меню** — на 4.3″ места хватает на видимую навигацию.
- **Не делать модалки внутри модалок** — confirm должен быть terminal, не открывать новые модалки.
- **Не показывать одновременно загрузку и контент** — если данные тянутся, **skeleton placeholder**, не «(loading…)» текстом.
- **Не использовать иконки без подписей в первой раз** — на главных вкладках 🏠 📁 💾 📊 ⚙ имеют текст. Внутри страниц иконки могут быть без, потому что контекст ясен.
- **Не лить градиенты** — flat colors, чёткие границы.
- **Не делать toast'ы которые блокируют** — они **только информируют**, никаких confirm-action в toast'е.

## 9. Открытые вопросы по визуалу

- **Иконки** — emoji или собственный set (Lucide / Tabler)? Emoji проще но непоследовательно (на разных шрифтах разный вид). Lucide = единый стиль но Inline SVG зависит от шрифтового рендеринга.
- **Анимации** — animation-duration 200-300ms во всех transition'ах. Микро-spring на кнопках? Или строго ease-out cubic?
- **Light theme** нужен? Думаю нет — kiosk, чёрный экономит OLED, в темноте читается лучше. Но многие любят. Опция в Settings — 2× работы по дизайну. Решим позже.
- **Кастомные шрифты** — Inter ~200KB надо загрузить. Локально через `@font-face` чтобы не тянуть с интернета (offline-first kiosk).
- **Точки vs иконки** для статуса — точка лаконичнее, иконка ✓/⚠/✗ информативнее. Думаю **точка + текстовая метка**.
